📜 Formal Methodology & Data Privacy Framework

1. Computational Methodology

The Political Pulse platform employs a Deterministic Data Aggregation model. Unlike traditional polling, this system utilizes 1:1 token mapping to ensure statistical accuracy. The backend logic follows a three-tier architecture:

  • Ingestion Layer: Captures user intent through a sanitized input field, filtering for the @gmail.com top-level domain to mitigate bot infiltration.
  • Processing Layer: Uses a linear summation algorithm to calculate vote weights. The "AI Prediction" module utilizes Heuristic Analysis to project the current leader based on the Delta (difference) between total ballots cast.
  • Presentation Layer: Implements the Chart.js API to render an asynchronous Scalable Vector Graphic (SVG) representation of the data.

2. Integrity & Anti-Fraud Protocols

To preserve the sanctity of the democratic process, the system enforces Strict Single-Entry Policy (SSEP). Upon successful ballot submission, the user's unique Gmail identifier is appended to an encrypted array within the client-side metadata. Any secondary attempt to access the voting function with the same identifier triggers an immediate 403-style interception, blocking the duplicate entry before it reaches the data state.

3. Privacy & Compliance Statement

This application is designed in alignment with General Data Protection Regulation (GDPR) principles for localized data handling:

  • No External Transmission: To ensure zero-leakage, no data is sent to external servers or third-party APIs. All information remains in LocalStorage.
  • Right to Erasure: Users have the right to purge their data. The Administrator retains the authority to execute a "Global Hard Reset," which clears the pollData cache and restores the system to its factory default state.
  • Anonymity: Voter identities are used solely for validation and are never displayed on the public-facing dashboard.

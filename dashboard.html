📜 System Methodology & Philippine Data Privacy Compliance

I. Technical Methodology

The Political Pulse v2.0 system architecture is built upon a Client-Side Data Processing Model. This methodology ensures zero-latency updates and high availability for localized polling. The system workflow is categorized into four distinct phases:

  • 1. Identity Verification Layer: The system utilizes a restrictive domain-validation algorithm that filters entries based on the @gmail.com Top-Level Domain (TLD). This acts as a primary defense against automated bot submissions.
  • 2. Algorithmic Processing: Votes are processed via an asynchronous JavaScript execution loop. The AI Prediction Module performs a real-time delta analysis between candidates to determine statistical leads and probability outcomes.
  • 3. Data Persistence (No-SQL Simulation): Utilizing window.localStorage, the system simulates a No-SQL database environment. This allows for persistent state management without the overhead of external server calls, ensuring the application remains functional in offline-first environments.
  • 4. Dynamic Visualization: Data is rendered through the Chart.js API, mapping internal arrays to a 2D canvas element for clear, academic-level data representation.

II. Privacy Policy (RA 10173 Compliance)

In accordance with the Data Privacy Act of 2012 (Republic Act No. 10173) of the Philippines and the guidelines set by the National Privacy Commission (NPC), this project adheres to the principles of Transparency, Legitimate Purpose, and Proportionality.

1. Data Collection & Purpose

We collect limited Personal Information (PI) consisting of your Gmail address and a system-generated timestamp. The sole purpose of this collection is to prevent "Double Voting" and ensure the integrity of the polling results for academic research and demonstration purposes.

2. Data Retention and Disposal

Consistent with the DPA, data is only kept for the duration of the project demonstration. Since all data is stored in the user's LocalStorage, the data subject (the voter) maintains full control and may dispose of their records at any time by clearing their browser cache or via the Administrator’s "Global Reset" function.

3. Security Measures

Technical and organizational security measures are implemented to protect the data against natural and human dangers. All email addresses are stored in a private array and are not visible to other voters on the public dashboard, maintaining the Secret Ballot principle.

4. Rights of the Data Subject

Under RA 10173, participants have the right to: (a) Be informed that their data is being processed; (b) Object to the processing; (c) Access their data; and (d) Rectify or erase their data. By proceeding with the vote, the user grants "Express Consent" for this specific academic exercise.
